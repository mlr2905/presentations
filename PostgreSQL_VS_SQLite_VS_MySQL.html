PostgreSQL VS SQLite VS MySQL

Data Manipulation
Insert Record
INSERT INTO users (id, name, email) VALUES (1, 'John Doe', 'john@example.com');
Update Record
UPDATE users SET email='john.doe@example.com' WHERE id=1;
Delete Record
DELETE FROM users WHERE id=1;
Search Record by ID
SELECT * FROM users WHERE id=1;
Data Retrieval
Join Tables
SELECT users.name, orders.order_id, orders.amount FROM users INNER JOIN orders ON users.id = orders.user_id;
Count Records
SELECT COUNT(*) FROM users;
Average Value
SELECT AVG(amount) FROM orders;
Maximum Value
SELECT MAX(amount) FROM orders;
Functions
Function in MySQL
-- Create function to get username by ID CREATE FUNCTION get_username_by_id(user_id INT) RETURNS VARCHAR(255) BEGIN DECLARE username VARCHAR(255); SELECT name INTO username FROM users WHERE id = user_id; RETURN username; DELIMITER ;
Function in PostgreSQL
CREATE OR REPLACE FUNCTION calculate_tax(amount numeric) RETURNS numeric AS $$ DECLARE tax numeric; BEGIN tax := amount * 0.2; -- 20% tax rate RETURN tax; END; $$ LANGUAGE plpgsql;
Function in SQLite
CREATE FUNCTION max_value() RETURNS INTEGER AS $$ SELECT MAX(value) FROM data; $$;
Triggers
Trigger in MySQL
-- Create trigger to update last_modified field on update CREATE TRIGGER update_last_modified_trigger BEFORE UPDATE ON users FOR EACH ROW SET NEW.last_modified = CURRENT_TIMESTAMP;
Trigger in PostgreSQL
CREATE OR REPLACE FUNCTION update_last_modified() RETURNS TRIGGER AS $$ BEGIN NEW.last_modified = CURRENT_TIMESTAMP; RETURN NEW; END; $$ LANGUAGE plpgsql; CREATE TRIGGER update_last_modified_trigger BEFORE UPDATE ON users FOR EACH ROW EXECUTE FUNCTION update_last_modified();
Trigger in SQLite
CREATE TRIGGER update_last_modified AFTER UPDATE ON users BEGIN UPDATE users SET last_modified = CURRENT_TIMESTAMP WHERE id = NEW.id; END;

Language Differences

MySQL

MySQL uses DELIMITER to define the start and end of stored procedures and functions.

MySQL triggers can be defined to execute before or after an INSERT, UPDATE, or DELETE operation.

MySQL supports non-transactional storage engines like MyISAM, in addition to transactional engines.

PostgreSQL

PostgreSQL uses LANGUAGE plpgsql for stored procedures and functions written in PL/pgSQL.

PostgreSQL triggers can be defined to execute before or after an INSERT, UPDATE, or DELETE operation.

PostgreSQL supports advanced data types like JSON, arrays, and full-text search capabilities.

SQLite

SQLite does not support stored procedures like MySQL or PostgreSQL, but it does support user-defined functions (UDFs).

SQLite triggers can be defined to execute before or after an INSERT, UPDATE, or DELETE operation.

SQLite is a self-contained, serverless, zero-configuration SQL database engine.
